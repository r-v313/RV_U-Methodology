58 DoS Techniques ▼
⚠️
Only test DoS vulnerabilities on programs that explicitly allow it. Never perform actual DoS attacks.
  • ReDoS: Supply regex-triggering input to slow regex operations. Test with long strings of repeating characters.
  • Large file upload: Upload max-size files to consume storage/bandwidth
  • Zip bomb: Upload a compressed file that expands to huge size
  • XML bomb (Billion Laughs): Deeply nested XML entities
  • Algorithmic complexity: Trigger O(n²) or worse operations
  • Pixel flood: Upload 1x1 PNG claiming to be 50000x50000
  • Long password DoS: bcrypt slows with very long passwords — send 1MB password
  • Rate limiting absence: Demonstrate missing rate limits without actually causing DoS
